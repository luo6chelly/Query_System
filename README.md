# 东莞理工学院新生入学 RAG 智能问答系统
## 项目简介
本项目是《云计算与大数据应用开发》课程期末大作业，基于 Dify 平台搭建面向东莞理工学院新生的 RAG（检索增强生成）智能问答系统。

项目将**新生报到、后勤服务、宿舍管理、校园卡、学费缴纳**五大类官方文档构建向量知识库，新生通过自然语言提问，快速获取权威、准确的入学解答。

系统基于 Docker 容器化部署，前端采用 Svelte + Vite 开发，以学校吉祥物「提蜜蜜 Timi」作为核心视觉形象，同时适配 PC 端与移动端，完整覆盖课程要求的云计算、大数据、AI 智能体、Web 应用开发核心技术点。

## 技术栈
### 云原生与部署层
- 容器编排：Docker + Docker Compose 全链路一键部署
- AI 开发平台：Dify 开源智能体开发平台
- 底层存储组件：
  - PostgreSQL 15：业务主数据库
  - Redis 6：缓存、消息队列
  - Weaviate：向量数据库，用于文档语义检索

### AI 核心层
- 核心方案：RAG 检索增强生成、混合检索（语义+关键词）
- 大语言模型：qwen2.5:7b
- 知识库能力：PDF 自动分片、向量化存储、溯源引用

### 前端应用层
- 开发框架：Svelte + Vite
- 页面能力：响应式布局，PC / 移动端双端自适应
- 交互设计：对话聊天界面 + 侧边历史会话记录

## 系统架构
### 1. RAG 智能体工作流（两段式检索生成链路）
1. 用户输入：接收用户自然语言入学提问
2. 知识检索：绑定东莞理工新生专属知识库，混合检索召回相关文档片段
3. LLM 生成：检索片段作为上下文输入 qwen2.5:7b，搭配专属新生提示词生成回答
4. 结果返回：输出问答内容+官方文档引用来源至前端界面

### 2. 整体分层架构
```plaintext
前端交互层（Svelte H5 对话页面）
        ↓
Dify 智能体服务层（API / Worker / Web 控制台）
        ↓
数据存储层（PostgreSQL + Redis + Weaviate 向量库）
        ↓
容器编排层（Docker Compose 统一管理所有服务）
```

## 部署与启动指南
### 一、后端 Dify 平台部署
#### 1. 环境前置要求
- 已安装 Docker + Docker Compose V2
- 硬件建议：2 核 CPU + 4GB 以上内存 + 30GB 空闲磁盘
- 自备通义千问 qwen2.5:7b 模型 API 密钥

#### 2. 启动容器服务
```bash
# 进入部署目录
cd deploy

# 修改 .env 文件，自定义数据库密码、端口映射等配置

# 后台一键启动全套容器服务
docker compose up -d
```
执行后自动启动 API、Web、Worker、PostgreSQL、Redis、Weaviate 全部组件。

#### 3. 知识库与智能体工作流配置
1. 浏览器访问 Dify 控制台，注册初始化管理员账号
2. 知识库模块上传 `knowledge-base/` 下全部 PDF，自动完成文本分片、向量索引构建
3. 创建应用，编排完整 RAG 工作流：
   - 用户输入节点：接收前端提问
   - 知识检索节点：绑定新生知识库，配置混合检索、相似度阈值、召回条数
   - LLM 大模型节点：选择 qwen2.5:7b，配置新生问答专属提示词，注入检索上下文
   - 直接回复节点：输出带引用来源的回答
4. 发布应用，复制接口地址、API Key，供前端调用

### 二、前端项目启动
#### 1. 环境前置要求
Node.js 16 + npm

#### 2. 安装依赖
```bash
cd frontend
npm install
```

#### 3. 接口配置
在前端配置文件填入 Dify 应用 API 地址与访问密钥。

#### 4. 本地开发调试
```bash
npm run dev
```
运行后打开本地链接即可预览对话页面。

#### 5. 生产环境打包部署
```bash
npm run build
```
打包产物输出至 `dist` 文件夹，可部署任意静态网页服务器。

## 核心功能
- **全场景入学问答**：覆盖报到、后勤、宿舍、校园卡、缴费五大场景，所有回答溯源官方PDF，自动标注文档来源，信息权威可查。
- **双端自适应界面**：响应式页面，PC大屏、手机竖屏完美适配；移动端侧边栏一键查看历史对话。
- **校园特色视觉**：以东莞理工吉祥物「提蜜蜜 Timi」为UI核心，贴合校园新生场景，提升使用亲切感。
- **多会话历史留存**：自动保存所有对话记录，支持新建、切换、回看历史问答。
